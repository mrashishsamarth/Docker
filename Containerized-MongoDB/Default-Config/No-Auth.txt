The default configuration ususally does not have authorization enabled, meaning once you have the container up and running, you can connect to it and access the data.Apart from the authorization, the default installation does not have any usable databases as well.

The default port for mongod and mongos instances is 27017
The default port for mongod when running with --shardsvr command-line option or the shardsvr value for the clusterRole setting in a configuration file is 27018
The default port for mongod when running with --configsvr command-line option or the configsvr value for the clusterRole setting in a configuration file is 27019

The ports can be modified.
